What We Build

Local-first storage: Photo saved to device immediately on capture. Upload is a background job, not a foreground operation. User can close the app ‚Äî photo will upload when connectivity allows.
Background queue: WorkManager (Android) / BGTaskScheduler (iOS) manages upload queue. Exponential backoff retry on failure. Server-side confirmation ID stored locally ‚Äî we know exactly which photos are synced.
Zero data loss guarantee: Photos are only removed from local storage after server confirmation. "Randomly disappears" bug is structurally impossible in the new architecture.
