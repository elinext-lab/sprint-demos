Crash-on-Task: Main Thread Memory Pressure

The new "Field Service Your Way" app (419 MB) loads the full job JSON payload synchronously on the main thread. When a technician taps "Complete", the app triggers both a UI state change and a background sync simultaneously ‚Äî iOS kills the process under memory pressure.

// BROKEN ‚Äî sync on main thread func completeJob(_ job: Job) { updateUI(status: .complete) // ‚Üê main thread syncToServer(job) // ‚Üê blocks same thread }
‚úÖ Fix

Offload sync to a background `DispatchQueue`. Add crash recovery that restores unsaved state on next launch.

// FIXED ‚Äî async sync func completeJob(_ job: Job) { updateUI(status: .complete) DispatchQueue.global(qos: .utility).async { syncToServer(job) // background thread } }

Ghost Input: iOS Keyboard Event Collision

iOS fires UIKeyboardWillShowNotification in a way that interacts incorrectly with the cross-platform KeyboardAvoidingView behavior. When the keyboard animates in, the component remounts and resets the controlled input state ‚Äî clearing whatever was typed.

// BROKEN ‚Äî generic cross-platform config <KeyboardAvoidingView behavior="padding" // ‚Üê breaks on iOS 16+ keyboardVerticalOffset={0} > <SearchInput value={query} /> </KeyboardAvoidingView>
‚úÖ Fix

Use iOS-specific KeyboardAwareScrollView from react-native-keyboard-aware-scroll-view. Preserve input state via useRef rather than remounting on keyboard events.

// FIXED ‚Äî iOS-specific handling <KeyboardAwareScrollView enableOnAndroid={false} // iOS only extraScrollHeight={20} keyboardShouldPersistTaps="handled" > <SearchInput ref={inputRef} /> // stable ref </KeyboardAwareScrollView>

Sync Lag: iOS Background Refresh Throttling

iOS aggressively throttles background app refresh based on battery, usage patterns, and system load. Without explicit BGTaskScheduler configuration and push-triggered silent notifications, the app can only sync when foregrounded ‚Äî leaving technicians with stale data for 10‚Äì15 minutes at a time.

Android's WorkManager / JobScheduler is far more permissive about background execution, which is why Android technicians see near-real-time updates while iOS technicians see stale data.

// BROKEN ‚Äî sync only on foreground AppState.addEventListener('change', state => { if (state === 'active') syncJobs() }) // ‚Üê iOS background: nothing happens
‚úÖ Fix

Register a BGAppRefreshTask with iOS 13+ BGTaskScheduler. Configure APNs silent push notifications to trigger a background sync on job assignment events ‚Äî no foreground required.

// FIXED ‚Äî background fetch + silent push BGTaskScheduler.register( forTaskWithIdentifier: "co.zuper.sync", using: nil ) { task in handleSyncTask(task as! BGAppRefreshTask) } // APNs silent push ‚Üí wake + sync in <30s
