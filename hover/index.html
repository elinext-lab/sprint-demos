Technical root cause
iOS capture is native. Android is not.
This is the structural reason the gap exists ‚Äî and why it won't close without a dedicated Android engineering pass.
‚úì iOS ‚Äî Native SDK (hover-capture-ios)

github.com/hoverinc/hover-capture-ios

// Native AVFoundation ¬∑ ARKit access // Direct camera2-equivalent on iPhone // Consistent across all iOS hardware // EXIF metadata stable ¬∑ orientation reliable

Apple's unified camera API + iOS process management = predictable capture behavior across all iPhone models. HOVER already solved this for iOS.

‚ö† Android ‚Äî React Native Bridge

No hover-capture-android in hoverinc GitHub org

// RN bridge ‚Üí camera2 API // Samsung One UI: aggressive bg kill // Xiaomi MIUI: camera API deviations // EXIF orientation inconsistent by OEM

Android device fragmentation: hundreds of OEM configurations. Camera2 API behavior differs by device, OS version, and OEM skin. The RN bridge can't abstract all of this cleanly.
