Business Impact
Construction sites, warehouse floors, rural hospitality venues — connectivity drops are not edge cases. When clock-in fails silently, the shift goes unrecorded entirely. This is not a null value in your dataset; it is an absent record — far harder for Deputy AI to detect and compensate for than a visible error state.
Technical Approach

Room database write queue — clock events written locally before any network call

WorkManager background sync — queued events flushed when connectivity restores, with exponential backoff

Same API endpoints — no backend schema changes required; payload structure unchanged

Matches iPad Kiosk behavior — the offline model is already proven inside the Deputy product family

Business Impact
Incorrect mid-shift clock-outs require manual employer corrections on a weekly cadence. In Australia, this creates Fair Work Act exposure on disputed timesheet records. In the US, FLSA requires accurate time records — systematic errors during litigation are a material liability. More directly: every auto clock-out injects a false shift-end event into the dataset Deputy AI learns from.
Technical Approach

Background URLSession — keeps session heartbeat active when app moves to background or screen locks during shift

Grace period before clock-out — foreground interruption triggers grace window, not immediate termination; clock-out requires explicit worker confirmation

CLLocationManager background mode — significant-location-change monitoring keeps process alive without battery penalty

Local session token persistence — session state written to Keychain; survives iOS process termination events
