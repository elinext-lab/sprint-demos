Documented Failure Analysis

4 Failure Categories.
All Reproducible. All Fixable.

These aren't anecdotes. They appear consistently across Play Store reviews, your support documentation, and your own developer responses in the store.

🔔
Failure 01 — Notification Routing

FCM Push Notification Routing Broken

"receive messages I can't read sometimes because app simply doesn't open" Play Store review — verified user
Root Cause FCM data payload delivered without back-stack Intent routing. App launches to home screen instead of message thread. NavDeepLinkBuilder or PendingIntent with explicit TaskStackBuilder not wired to notification click handler.
Evidence Vivian maintains a dedicated support article for notification troubleshooting — a live signal that engineering has not resolved this at the source.
Impact Messaging is the primary recruiter-nurse touchpoint. Broken notification routing directly reduces platform-mediated job matches.
🐞
Failure 02 — Force-Close Crash

Job Browse Crash / Force-Close

"now I can't even browse jobs. It just shuts Vivian down." Play Store review — verified user
Root Cause RecyclerView adapter NPE or OOM on paginated job lists. Likely missing null safety on async data binding, or unbounded pagination loading all items into memory. Paging 3 library adoption would eliminate both.
Context Crash occurs at highest-intent moment in the user journey — actively browsing jobs. Losing a user here is losing a placement.
Impact Direct revenue impact. A nurse who can't browse jobs cannot book a shift. Core funnel failure.
⌨
Failure 03 — Samsung Keyboard Freeze

Samsung Keyboard Freezes Entire App

"We identified an issue with the Samsung Android keyboard freezing multiple apps — we recommend using a different keyboard as a workaround." Vivian Health developer response — Play Store
Root Cause IME/InputMethodManager lifecycle failure in custom text inputs. Samsung keyboard (SwiftKey variant) triggers edge-case in window focus change callbacks. Proper IME contract implementation resolves this without requiring users to change their keyboard.
Scale Samsung holds ~30% of US Android market share. "Use a different keyboard" is not a fix — it's a churn trigger.
Impact 30% of Android users (Samsung) encounter this in any text input flow — profile setup, messaging, job search. Blocks core interactions.
🔁
Failure 04 — Navigation State Loss

Filter / City State Lost on Back Navigation

"lists of cities are reset when you click on one and then go back, with no way to filter by what cities have jobs." Play Store review — verified user
Root Cause Fragment state not preserved across back-stack. Filter selections held in local state instead of ViewModel + SavedStateHandle. Navigation Component + SavedStateHandle eliminates this class of bug entirely.
Frequency Job search with location filters is the most common user workflow. This bug affects every session that includes city-based filtering.
Impact Repeated state resets force nurses to abandon search flows. Increases bounce rate on job discovery.
