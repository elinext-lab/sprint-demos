Regulatory context

Technical root cause analysis
Why these failures persist
Each failure category maps to a specific engineering root cause with a known fix.

GPS Cold Start Latency

The app requests GPS only at clock-in -- a point-in-time fix. No location pre-warming during transit. Caregiver arrives, taps clock-in, gets a stale Wi-Fi-triangulated location with 50-100m accuracy. Proximity geofence rejects it.

// Point-in-time GPS fix -- no pre-warming locationManager.requestSingleUpdate() // Returns cached/Wi-Fi location, not GPS fix

Data Sync Loss

Nurse documentation held in memory, not committed to local database before sync. When connectivity drops (common in homes -- basements, rural), the form submission fails and all entered data is lost.

// Direct server submission -- no local persistence api.submitNotes(formData) // fails silently on network loss // No Room/CoreData write-first architecture

Phantom Clock State

App shows local "clocked out" before server confirmation. If network request fails, server still shows caregiver clocked in. No state reconciliation on app reopen. Creates billing discrepancies and phantom overtime.

// Optimistic UI without server confirmation localState.setClockOut() // shown to user api.clockOut() // may fail silently

Auth Token Persistence

"Remember me" does not survive app restarts. Token not stored in Android Keystore or iOS Keychain. Device timestamp mismatch (acknowledged in Zendesk) causes token validation failure. Session timeout shorter than visit duration.

// Token in volatile storage -- lost on restart // No EncryptedSharedPreferences / Keychain // Clock-dependent expiry validation
The fix: GPS clock-in
From "Not Within Proximities" to reliable EVV verification
A foreground service pre-warms GPS during transit. By the time the caregiver arrives and taps clock-in, the GPS receiver has a precise fix.

Accuracy threshold check

The fix: nurse documentation
From "notes disappeared" to write-first architecture
Nurse notes write to encrypted local storage first. Sync to server is asynchronous with retry. No data is ever lost.
