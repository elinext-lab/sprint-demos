Evidence

  • Google Play user review: "5 to 10 minutes to submit progress notes even after recent updates" (confirmed)
  • Skedulo v3.16.1 hotfix (~May 2025): "key performance improvement released to address the Job notes issue" ‚Äî emergency release immediately after v3.16.0
  • NDIS compliance requirement: care workers must document visits before leaving client homes (NDIS Commission standards)
Root cause: The app waits synchronously for server acknowledgment before releasing the UI. When Salesforce API calls experience rate-limiting (confirmed in v3.21.1 release notes), the entire submission queue stalls. Fix: implement offline-first queue with optimistic local save ‚Äî worker sees "saved" immediately, background sync completes independently.

Evidence

  • App Store version history: "Resolved the app becoming non-responsive when uploading 10+ photos on iOS" ‚Äî December 10, 2025
  • Pattern: bug existed from October 2023 launch until December 2025 ‚Äî over 2 years undetected in QA (test matrix used 1-3 photos)
  • Android risk: the iOS fix may not have been mirrored to the Android implementation ‚Äî same memory loading patterns would cause same crash
Root cause: All UIImages loaded into RAM simultaneously before upload begins. On iPhone SE and iPhone 12 (common among care workers), 10+ full-resolution photos exhaust available memory. Fix: PHAsset lazy loading ‚Üí stream each image as it's needed, parallel uploads with concurrency limit of 3, background URLSession for resilience.

Evidence

  • iOS App Store review (Sep 2, 2024, 2‚òÖ): "Too many glitches! Doesn't update and logs you off so often."
  • Skedulo Plus troubleshooting documentation (live, confirmed): "Users being unexpectedly logged out during poor internet conditions."
  • Pre-research reports: workers reinstalling the app daily to resolve "page expired" errors ‚Äî workaround, not fix
Root cause: OAuth access tokens expire during offline periods. When connectivity returns, the app can't refresh the token (refresh token not persisted in Keychain, or expired simultaneously). Logout is the fallback. Fix: store refresh tokens in iOS Keychain / Android Keystore with proper encryption; implement silent background token refresh with exponential backoff; offline session continuation with local data persistence.
